# 我比AI强在哪？

| 想起了村上春树的《当我跑步时，我在想什么》，所以就按照热身、跑步、拉伸来叙述吧。

AI是一个极好的执行者，你只需要准确的告诉它你的需求，他通常都能超出预期的完成你布置的任务，所以有了AI又要我何用？

## 热身

用AI前你通常要想清楚你要什么。比如你要一个网站，这个网站是做什么的？有什么风格？实现那些细节？达成什么目的？总之，此时的AI替代了你打工时的角色。如果不知道如何写提示词，不妨想想，你在接手一个新项目的时候会问什么，比如：目标是什么？技术要求是什么？有什么现有的资源？你所有需要的知道的，AI也同样需要知道。

不仅写代码如此，前些天试了试AI写网络小说，随便找个命题丢给它，结果完全不是我想要的样子。当然，这不怪它，我没有设定任务，没有背景，没有主要剧情冲突，总之写小说该有的一切我都没有，又怎么好苛求它交出一份好的答卷（不过某些傻逼老板可就不是这么想了）。

总之，在打开一个对话框前，你要有充足的腹稿，清晰的目标，才能得到想要的答案。这不是件坏事，至少表明了你还有决定权，相比AI会多一些能动性和目标。

## 跑步

如果上一个阶段你没有明确的结果，最好不要开始这一阶段的工作。因为你会发现处处不满意，怎么改都改不对，不如推翻了重写。

当然如果你已经有了明确的腹稿，这一阶段大概率是能够得到你想要的结果的。可这就够了么？不知道你有没有过一种体验，在做事情的过程中，看到的细节越来越多，想法也越来越多，思路越来越清晰，对当前事情的认识也开始变得深刻。于是不断引入新的想法，或者开启新的项目。

写作也是如此，我想写点什么，开始只有几句话想说，说着说着想法就多了，不断地梳理自己的思路，将碎片梳理成框架，有时候还会产生新的想法，积累成下一篇内容（某些挖坑博主大概也是这样（这么说的话，这种博主是不是洗清了AI写稿的嫌疑））。

所以多动动手是有好处的，如果实在没时间做，那至少多看看，多想想，积累些经验，积累些想法，这些看不见的东西才是你比AI强的地方。

## 拉伸

AI的代码是中用不中看的，AI写的文章读起来是别别愣愣的。相信我，至少今天，AI的屁股还得我们自己擦。那，你擦的明白么？代码框架的逻辑你懂么？环境千奇百怪的问题，你能定位么？退一步，你能说明白的问题么？文章没写好，你有能力改的精彩么？

AI是一个靠谱但能力有限的工具人，而我们是给它擦屁股的人，是给它兜底的人，所以，这个底你能兜得住么？

所以，软技能上我比AI会多些想法，多些能动性，我比AI更能和这个世界交互。硬技能上，我大部分是不如AI的，但我们能交流，偶尔我还比它强点，这大概就够用了。
